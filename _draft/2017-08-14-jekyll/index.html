jekyll과 주어진 테마를 이용해서 만든 Github 블로그에 Disqus 추가하기

August 14, 2017

Github은 저같은 풋내기 프로그래머들에게는 환상적인 공간입니다.

이번엔 블로그를 만들어 봤는데요. 적절한 한국어로 된 설명이 드물거나 찾기 힘든 것 같아서 포스팅 해봅니다.

특히 jekyll 언어를 충분히 공부하지 않은 상태에서 특정 테마를 다운받아서 사용할 경우에 경험할 수 있는 시행착오를 범하지 않도록 하는 방향입니다.

영어로 된 게시물들은 구글링으로 쉽게 얻을 수 있습니다.

이 포스팅 또한 jekyll 홈페이지와 Sechter의 블로그의 내용의 번역이 주가 될 예정이니 더 자세한 내용을 원하시면 해당 링크로 들어가서 확인하시기 바랍니다.

블로그를 스스로 제대로 만들어보겠다고 생각하신다면 jekyll 홈페이지를 많이 보세요. 설명도 쉽고, 자세하게 잘되어있습니다. 한글화가 안되어있는 부분이 조금 아쉬운데, 제대로 공부해보겠다고 생각하시는 분이 있으면 직접 번역 프로젝트를 실행해서 Github에 repository에 기여해보시는 것도 좋을 듯 합니다.

준비물

  • Linux나 macOS 가정하고 설명하겠습니다. 제가 macOS를 쓰거든요.

  • Ruby version 2.1 이상 (그냥 최신것 받으시면 됩니다.)

  • RubyGems

$ gem install jekyll

버전 체크는

$ jekyll -v
  • GCC/Make

테마를 다운 받기 전

만약 테마를 받기 전에 연습삼아 이미 블로그를 만들어 웹에 띄어본 상태라면, index.md 나 html 파일들이 새 테마와 충돌을 할 수도 있어요. gemfile 관련 파일과 이미 포스팅했던 포스트파일들을 남겨두고 삭제하거나 따로 옮겨둡니다.

테마를 다운 받은 후

테마는 대부분 Github을 통해 다운받게 됩니다. 그 이후 커스토마이징 하는데, _config.yml 등의 .yml 확장자를 가진 파일을 보면 됩니다. 특정 문법으로 씌여있는 것도 아니라서 그 안에 들어가서 본인 이름이나 블로그 이름같은 것을 바꿔 넣으면 됩니다. 배경 사진같은 것을 바꿀 때는 그 사진 사이즈에 맞게 리사이징을 해서 사진 이름을 바꿔넣으면 돼요.

disqus 설치

이부분도 테마에서 설정이 되어있을 수도 있어요. 우선 disqus 가입은 하셔야 하고요.

가입하고 블로그 이름이나 url 올리고 한 후에 차례 대로 embed복사해와서 disqus.html 같은 파일에 가져다 붙이고, layouts 폴더에 있는 post.html 도 수정한 이후에 s (이부분은 자세한 설명 생략할게요. 다른 여타 블로그들에 잘 나와있습니다. 여기 포스팅한 내용들은 제가 조금이라도 시행착오를 겪은 부분들입니다. )

일반 세팅으로 들어가시면, 사진처럼 Shortname이 나옵니다.

이걸 위에 복사해 넣는 코드에서 SHORTNAME부분 대신에 가져다 붙여야해요.

제 블로그의 테마에서는 위의 git commit history에 나와있는 것처럼(붉은 색 라인이 삭제되고 녹색 라인이 추가) shortname 에 disqus 사이트에서 정해준 physhiks-data-science 를 채워넣었습니다. _config.yml 과 _include 폴더에 들어있었던 disqus_comments.html 을 수정했네요. disqus 관련 파일이름은 테마에 따라 다를 수 있어요.

또한 YAML front matter block이라고

---


comments: true
---

이런 식으로 md파일이나 html 파일 등등에서 맨 위에 들어가는 것이 있거든요.

disqus댓글을 사용하고 싶은 페이지의 md, html파일을 편집해서 위에처럼 comments: true 로 바꿔주시면 됩니다.

그렇게 하고 제대로 작동하는지 확인해보세요. 작동법은 타 사이트들에 자세히 설명되어있으니 아무거나 보고 확인하시면 됩니다.

이미지

위에 이미지 올린 김에 어떻게 올라간지 간단히 볼까요.

이 포스팅은 마크다운 포맷으로 적고 있습니다. 사진도 그렇게 마크다운 문법으로 올리시면 돼요. 저는 블로그 파일들을 https://physhik.github.io 에서 관리하고 있으니까, 그 사이트 이름 뒤에 이미지가 들어있는 폴더 이름과 파일 이름을 가져다 붙인 겁니다. 제 블로그 파일 세팅은 이렇게 되어있지만, 따로 /asset/images/ 같은 폴더로 관리하는 경우엔 다르게 적어줘야 합니다.
